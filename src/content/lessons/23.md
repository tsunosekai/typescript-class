## 🎨 フロントエンド強化

今の知識をさらに深める技術です。

| 技術 | 難易度 | 説明 |
|------|--------|------|
| **Tailwind CSS** | ⭐ | クラス名だけでスタイル。今一番人気 |
| **React Router** | ⭐⭐ | 複数ページのアプリを作る |
| **TanStack Query** | ⭐⭐ | API通信を簡単・高速に |
| **Zustand** | ⭐⭐ | シンプルな状態管理 |
| **React Hook Form** | ⭐⭐ | フォームのバリデーション |

```mermaid
flowchart LR
    subgraph Style["見た目"]
        T["Tailwind CSS"]
    end
    subgraph Route["ルーティング"]
        R["React Router"]
    end
    subgraph Data["データ"]
        Q["TanStack Query"]
        Z["Zustand"]
    end
    subgraph Form["フォーム"]
        F["React Hook Form"]
    end

    T -.-> R -.-> Q -.-> Z -.-> F
```

**おすすめ順**: Tailwind → React Router → TanStack Query

## 🚀 フレームワーク

Reactをさらに便利にしたフレームワークです。

| 技術 | 難易度 | 説明 |
|------|--------|------|
| **Next.js** | ⭐⭐⭐ | React公式推奨。ルーティング・SSR標準搭載 |
| **Remix** | ⭐⭐⭐ | Web標準重視。フォーム処理が得意 |

```mermaid
flowchart TB
    REACT["React（今の知識）"]
    NEXT["Next.js"]
    REMIX["Remix"]

    REACT --> NEXT
    REACT --> REMIX

    NEXT -.- N1["App Router"]
    NEXT -.- N2["Server Components"]
    NEXT -.- N3["API Routes"]

    REMIX -.- R1["Loader / Action"]
    REMIX -.- R2["Form処理"]
```

**おすすめ**: **Next.js** が圧倒的に人気。求人も多い。

## ⚙️ バックエンド（サーバーサイド）

自分でAPIを作る技術です。

| 技術 | 難易度 | 説明 |
|------|--------|------|
| **Node.js** | ⭐⭐ | サーバーでJavaScript/TypeScriptを動かす |
| **Express** | ⭐⭐ | Node.js用Webフレームワークの定番 |
| **Hono** | ⭐⭐ | 軽量・高速。Cloudflare Workersと相性◎ |
| **Prisma** | ⭐⭐⭐ | データベースを簡単に扱えるORM |

```mermaid
flowchart TB
    NODE["Node.js<br/>サーバーサイドJS"]

    EXPRESS["Express<br/>定番フレームワーク"]
    HONO["Hono<br/>軽量・エッジ対応"]

    PRISMA["Prisma<br/>DB操作"]
    DRIZZLE["Drizzle<br/>軽量ORM"]

    NODE --> EXPRESS
    NODE --> HONO
    EXPRESS --> PRISMA
    HONO --> DRIZZLE
```

**おすすめ**: Cloudflareを使うなら **Hono**、それ以外は **Express**

## ☁️ BaaS（Backend as a Service）

バックエンドを自分で作らなくても使えるサービスです。

| サービス | 難易度 | 説明 |
|----------|--------|------|
| **Supabase** | ⭐⭐ | PostgreSQL + 認証 + ストレージ。オープンソース |
| **Firebase** | ⭐⭐ | Google製。リアルタイムDB・認証・ホスティング |

```mermaid
flowchart TB
    subgraph Supabase["Supabase"]
        S1["PostgreSQL<br/>（本格的なDB）"]
        S2["認証"]
        S3["ストレージ"]
        S4["Edge Functions"]
    end

    subgraph Firebase["Firebase"]
        F1["Firestore<br/>（NoSQL）"]
        F2["Authentication"]
        F3["Cloud Storage"]
        F4["Cloud Functions"]
    end
```

**どっちを選ぶ？**
- **Supabase**: SQLを学びたい、本格的なDB構造を作りたい
- **Firebase**: 手軽に始めたい、リアルタイム同期が必要

## 🌐 デプロイ（公開）

作ったアプリを世界に公開する場所です。

| サービス | 難易度 | 説明 |
|----------|--------|------|
| **Vercel** | ⭐ | Next.js開発元。デプロイ最簡単 |
| **Cloudflare Pages** | ⭐ | 無料枠が太っ腹。高速 |
| **Netlify** | ⭐ | 静的サイトに強い |

```mermaid
flowchart TB
    subgraph Vercel["Vercel"]
        V1["Next.js最適化"]
        V2["プレビュー環境"]
        V3["Edge Functions"]
    end

    subgraph Cloudflare["Cloudflare"]
        C1["Pages<br/>静的サイト"]
        C2["Workers<br/>エッジ関数"]
        C3["D1<br/>エッジDB"]
        C4["R2<br/>ストレージ"]
    end

    subgraph Netlify["Netlify"]
        N1["静的サイト"]
        N2["フォーム処理"]
        N3["Functions"]
    end
```

**おすすめ**:
- Next.jsを使うなら → **Vercel**
- 無料で色々やりたい → **Cloudflare**
- シンプルに公開したい → **Netlify**

## 🔐 認証

ログイン機能を実装する技術です。

| 技術 | 難易度 | 説明 |
|------|--------|------|
| **Auth.js** | ⭐⭐⭐ | 旧NextAuth.js。様々なプロバイダ対応 |
| **Clerk** | ⭐⭐ | UI付きで簡単。無料枠あり |
| **Supabase Auth** | ⭐⭐ | Supabase組み込み認証 |
| **Firebase Auth** | ⭐⭐ | Firebase組み込み認証 |

## 技術選定チートシート

| やりたいこと | おすすめ技術 |
|-------------|-------------|
| スタイルを効率的に書きたい | **Tailwind CSS** |
| 複数ページのアプリを作りたい | **Next.js** or **React Router** |
| APIからデータを取得したい | **TanStack Query** |
| ログイン機能をつけたい | **Supabase** or **Clerk** |
| データベースを使いたい | **Supabase** or **Firebase** |
| アプリを公開したい | **Vercel** or **Cloudflare** |
| 自分でAPIを作りたい | **Hono** or **Express** |
